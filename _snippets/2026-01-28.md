### 11:30 - Phase 1 Investigation: Shape Factor Analysis

**Experiment:** Instrumented H (shape factor) and Cf (skin friction) distribution at α=0°, 10°, 15°

**Key Findings:**

#### Bug 1: Delayed Transition (CRITICAL)
| Solver | Transition x/c | Notes |
|--------|---------------|-------|
| XFOIL | 0.0186 | Free transition triggered early |
| RustFoil | 0.065 | Much later, BL already separated |

**Root Cause:** At the leading edge, Rθ is very low (~40-100), but critical Rθ depends on Hk:
- For Hk = 2.2: Rcrit ≈ 4677 (no amplification at low Rθ)
- For Hk = 4.0: Rcrit ≈ 35 (amplification possible)

RustFoil keeps Hk ≈ 2.2 in early stations, so no amplification occurs.
XFOIL's first pass has higher Hk values, triggering earlier transition.

#### Bug 2: Inverse Mode htarg Formula
**Problem:** `htarg = prev.hk + 0.03 * dx / theta` produces huge values when θ is tiny

**Example at ibl=24:**
- prev.hk = 3.05, dx = 0.003, θ = 3.5e-5
- htarg = 3.05 + 0.03 × 0.003 / 3.5e-5 = **5.62** (should be ~3.8)

Inverse mode converges correctly to htarg, but htarg itself is wrong!

**Evidence from debug trace:**
```
ibl=24 Newton iterations:
  iter=4: delta_ue=-0.01, Hk=3.65 (inverse mode triggers)
  iter=7: converged, Hk=5.43 (wrong target achieved!)
```

### CRITICAL ROOT CAUSE IDENTIFIED (15:30)

**The Problem:** RustFoil does BL marching ONCE, then uses Jacobian-based updates for subsequent global iterations. XFOIL re-marches the BL at EVERY global iteration.

**Evidence:**
- XFOIL at α=15°: Stations 93-99 show "MRCHUE: Inverse mode... Hk = 2.500" during marching
- After multiple global iterations with Ue updates, final H = 3.033 (exceeds htmax!)
- RustFoil: ibl=93 appears only ONCE in MRCHUE events, H stays at 2.378

**Why this matters:**
- The BL-inviscid coupling is NONLINEAR
- As Ue changes due to mass defect feedback, H can grow beyond htmax
- Jacobian updates can't capture this - you must re-march!

**The Fix:**
RustFoil needs to re-march the full BL (call `march_fixed_ue`) at every global Newton iteration, not just update using the Jacobian.

### 18:00 - Wake Station Coordinate Bug Fixed

**Root Cause Found:**
The "170% H error" was actually a **comparison error** - we were comparing:
- XFOIL lower surface TE (H=4.912)
- RustFoil upper surface (H=1.827)

**Real Issue Identified:**
Wake stations were setting `x_coord` to wake arc length (> 1.0), making them appear as "surface stations past TE" in traces. This confused the analysis.

**Fix Implemented:**
Changed 1 line in `crates/rustfoil-coupling/src/wake.rs` (line 282):
```rust
// Before: station.x_coord = x_new;  // Wake arc length > 1.0
// After:  station.x_coord = 1.0;     // Clamp to TE for reporting
```

**Results:**

### 19:00 - Critical Ue Capping Bug Fixed

**7.3% H Error Investigation:**
Station-by-station comparison revealed transition was occurring 43% late:
- XFOIL: transition at x=0.0117
- RustFoil: transition at x=0.0167
- Impact: **+69% CD error** (0.01342 vs 0.00793)

**Root Cause:**
Typo in 3 test files capped Ue at 0.01 instead of ensuring minimum:
```rust
// WRONG:
let ue_stag = ue.abs().min(0.01);  // Caps at 0.01!

// CORRECT:
let ue_stag = ue.abs().max(0.01);  // Ensures minimum
```

This caused artificially low Rθ = θ × Ue × Re, delaying transition.

**Results After Fix:**
| Metric | XFOIL | Before | After | Improvement |
|--------|-------|--------|-------|-------------|
| Transition x | 0.0117 | 0.0167 (+43%) | 0.0133 (+14%) | 67% better |
| CD | 0.00793 | 0.01342 (+69%) | 0.01000 (+26%) | 62% better |
