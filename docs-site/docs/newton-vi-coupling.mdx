# Newton Viscous-Inviscid Coupling

This document describes the implementation of XFOIL's Newton viscous-inviscid coupling algorithm in RustFoil.

## Overview

The viscous-inviscid coupling is the core of XFOIL's boundary layer solver. It couples the boundary layer equations to the inviscid flow solution through the **mass defect** variable:

$$
\text{mass\_defect} = U_e \cdot \delta^*
$$

Where $U_e$ is the edge velocity and $\delta^*$ is the displacement thickness.

## Key Components

### 1. VTI Signs

The **VTI** (Viscous-Tangent-Inward) sign array handles the direction convention for upper and lower surfaces:

- Upper surface: `VTI = +1`
- Lower surface: `VTI = -1`

This affects the VM matrix construction:

```rust
// XFOIL formula
let u_m_j = -vti[i] * vti[j] * dij[(i, j)];
```

For single-surface mode, all VTI values are `+1.0`.

### 2. Forced Changes (DUE, DDS)

When there's a mismatch between the current edge velocity and what UESET would compute from the mass defect, **forced changes** are added to the Newton residuals:

```rust
// Mismatch: current Ue vs Ue computed from mass defect
let due = ue_current[i] - ue_from_mass[i];
let dds = -delta_star / ue * due;  // Induced delta_star change

// Add to residuals
rhs[i][k] += vs1_delta[k] * dds1 + vs2_delta[k] * dds2
           + vs1_ue[k] * due1 + vs2_ue[k] * due2;
```

### 3. VM Matrix

The VM matrix couples mass defect changes at all stations to the boundary layer equations:

$$
VM_{i,j,k} = VS2_{k,\delta^*} \cdot D_{2,M_j} + VS2_{k,U_e} \cdot U_{2,M_j}
           + VS1_{k,\delta^*} \cdot D_{1,M_j} + VS1_{k,U_e} \cdot U_{1,M_j}
$$

Where:
- $D_{M_j} = \partial \delta^* / \partial \text{mass}_j = 1/U_e$ for local station
- $U_{M_j} = \partial U_e / \partial \text{mass}_j = -VTI_i \cdot VTI_j \cdot DIJ_{i,j}$

### 4. BLSOLV Algorithm

The block-tridiagonal solver follows XFOIL's algorithm with a key difference: **row 3 (mass) is normalized by VM diagonal**, not VA:

```rust
// Forward sweep: normalize row 3 by VM(3,IV,IV)
let pivot = vm_mod[iv][iv][2];
let pivot_inv = 1.0 / pivot;

// Back substitution: use mass change to update upstream
for iv in (2..n).rev() {
    let mass_delta = solution[iv][2];
    for kv in (1..iv).rev() {
        for k in 0..3 {
            solution[kv][k] -= vm_mod[kv][iv][k] * mass_delta;
        }
    }
}
```

### 5. UPDATE Routine

The XFOIL-style update computes new edge velocities **before** applying any updates:

1. Compute new $U_e$ from proposed mass changes
2. Compute global relaxation factor from ALL changes
3. Apply ALL updates with common relaxation

```rust
// Step 1: Compute new Ue from proposed mass changes
let proposed_masses: Vec<f64> = stations.iter().zip(deltas.iter())
    .map(|(s, d)| s.mass_defect + d[2]).collect();
let new_ue = compute_ue_from_mass_vec(&proposed_masses, ue_inviscid, dij, vti);

// Step 2: Compute global relaxation
let mut rlx = 1.0;
for each station:
    let dn_theta = delta[1] / theta;
    let dn_mass = delta[2] / mass_defect;
    let dn_ue = (new_ue - u) / 0.25;
    // Limit relaxation based on max change

// Step 3: Apply all updates
for each station:
    theta += rlx * delta[1];
    mass_defect += rlx * delta[2];
    u += rlx * (new_ue - u);
    delta_star = mass_defect / u;
```

### Critical Finding: BL Re-March Requirement (2026-01-28)

**Problem discovered**: RustFoil was marching the boundary layer ONCE before the Newton loop, then using only Jacobian updates for subsequent iterations. XFOIL re-marches the BL at EVERY iteration.

**Why this matters**:
- The Ue → H relationship is highly **nonlinear** near separation
- Jacobian updates assume linearity and cannot capture H exceeding htmax
- At high α, XFOIL's H grows from 2.5 → 3.0+ over iterations (nonlinear)
- RustFoil's H stayed capped at 2.4 using Jacobian-only approach

**Evidence**:
- XFOIL: Each station appears in MRCHUE events 10-20 times (re-marched each iteration)
- RustFoil (before fix): Each station appeared once (single march)

**Fix implemented** (viscal.rs lines 827-910):
```rust
for iter in 0..max_newton_iter {
    // Extract current Ue from stations (updated by previous iteration)
    let current_upper_ue: Vec<f64> = upper_stations.iter().map(|s| s.u).collect();
    let current_lower_ue: Vec<f64> = lower_stations.iter().map(|s| s.u).collect();
    
    // Re-march boundary layers with current Ue
    let upper_result = march_surface(&upper_arc, &current_upper_ue, re, msq, &march_config, 1);
    let lower_result = march_surface(&lower_arc, &current_lower_ue, re, msq, &march_config, 2);
    
    // Copy march results back to stations
    // (theta, delta_star, h, hk, cf, ctau, ampl, flow type, mass_defect)
    
    // Then proceed with Jacobian-based VI coupling correction
    // ...
}
```

**Impact**: Enables nonlinear H growth necessary for stall prediction, but exposed that underlying BL accuracy needs improvement (CD errors 30-120%).


#### Transition Theta Error (13.5%)

The theta error at the first turbulent station is caused by **missing TRDIF implementation**.

**Current behavior:**
- RustFoil uses pure turbulent `bldif()` for the entire X1→X2 interval at transition
- Error: 13.5% at transition station, diminishing to 3-4% downstream

**XFOIL behavior (TRDIF):**
- Splits interval at transition point XT
- Laminar equations: X1 → XT
- Turbulent equations: XT → X2  
- Combined residuals and Jacobians with chain rule through interpolated variables

**Current accuracy:**

| Metric | Error | Status |
|--------|-------|--------|
| Transition location | 0.1% | ✅ Excellent |
| Theta at transition | 13.5% | ⚠️ Known limitation |
| Theta downstream | 3-4% | ✅ Good |
| ctau at transition | 1.9% | ✅ Close to target |

### TRDIF Algorithm (TODO)

XFOIL's TRDIF (`xblsys.f` lines 1224-1578) handles the transition interval:

```text
1. Save state (COM1, COM2)
2. Compute weighting factors WF1, WF2 from XT derivatives (from TRCHEK2)
3. Interpolate variables to XT: TT, DT, UT with full derivatives

LAMINAR PART (X1 → XT):
4. Set "2" variables = interpolated "T" variables
5. Call BLKIN, BLVAR(1), BLMID(1), BLDIF(1)
6. Transform Jacobian: BL1, BL2 using chain rule through TT_*, DT_*, UT_*, XT_*

TURBULENT PART (XT → X2):
7. Compute initial ctau: ST = CTR * CQ where CTR = 1.8 * exp(-3.3/(Hk-1))
8. Compute ST derivatives through CQ and Hk
9. Set "1" = "T" variables, restore "2"
10. Call BLMID(2), BLDIF(2)
11. Transform Jacobian: BT1, BT2 using chain rule through ST_*, TT_*, DT_*, UT_*, XT_*

COMBINE:
12. VSREZ(1) = BTREZ(1)                    // Shear-lag: turbulent only
13. VSREZ(2:3) = BLREZ(2:3) + BTREZ(2:3)   // Momentum, shape: sum
14. VS1 = BL1 + BT1, VS2 = BL2 + BT2       // Jacobians: sum
15. Restore COM1
```

**Implementation complexity:** ~600 lines of Rust code including:
- Extended TRCHEK2 returning XT derivatives
- Full interpolation derivative tracking
- Chain rule transformations for both Jacobian blocks
