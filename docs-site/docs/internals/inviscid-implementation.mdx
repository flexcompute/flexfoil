# Inviscid Solver Implementation

This document describes the implementation of the inviscid panel method solver in the `rustfoil-inviscid` crate.

## Overview

The inviscid solver implements XFOIL's linear vorticity panel method for calculating the potential flow around airfoils. The implementation is a direct port of XFOIL's FORTRAN subroutines to ensure matching results.

## Architecture

```
rustfoil-inviscid/
├── src/
│   ├── lib.rs          # Main entry point, InviscidSolver struct
│   ├── geometry.rs     # NCALC, APCALC, TE geometry (AirfoilGeometry)
│   ├── influence.rs    # PSILIN - influence coefficient calculation
│   ├── system.rs       # GGCALC - matrix assembly and factorization
│   ├── solution.rs     # SPECAL/CLCALC - solution combination
│   ├── stagnation.rs   # STFIND - stagnation point detection
│   └── error.rs        # Error types
└── tests/
    └── xfoil_comparison.rs  # Validation against XFOIL FORTRAN
```

## Key Components

### 1. Geometry Processing (`geometry.rs`)

Matches XFOIL's `NCALC` and `APCALC` subroutines:

- **Arc length computation**: Cumulative distance along airfoil surface
- **Hermite cubic splines**: Using `SEGSPL` for smooth derivatives
- **Normal vectors**: Outward-pointing unit normals at each node
- **Panel angles**: Angle of each panel relative to horizontal
- **Trailing edge geometry**: `ANTE`, `ASTE`, `DSTE`, `SHARP` flags

```rust
let geom = AirfoilGeometry::from_points(&points)?;
// geom.nx, geom.ny: normal vectors
// geom.apanel: panel angles
// geom.sharp: true if TE is sharp
```

### 2. Influence Coefficients (`influence.rs`)

Matches XFOIL's `PSILIN` subroutine using the PSIS/PSID formulation:

The stream function is computed as:

```
ψ = (1/(4π)) Σ_j [ PSIS_j (γ_j + γ_{j+1}) + PSID_j (γ_{j+1} - γ_j) ]
```

In LaTeX notation: `ψ = (1/(4π)) Σ_j [ PSIS_j (γ_j + γ_{j+1}) + PSID_j (γ_{j+1} - γ_j) ]`

where the braces `{` and `}` are LaTeX grouping, not JSX expressions.

The influence coefficients are:
- **PSIS**: Sum term for average vorticity
- **PSID**: Difference term for vorticity gradient

```rust
let result = psilin(&geom, i, xi, yi);
// result.dzdg[j]: ∂ψ/∂γ_j at point (xi, yi)
```

### 3. System Assembly (`system.rs`)

Matches XFOIL's `GGCALC` subroutine:

1. Builds the $(N+1) \times (N+1)$ influence matrix $A$
2. Sets up boundary condition: ψ_induced = ψ₀ on surface
3. Applies Kutta condition: $\gamma_1 + \gamma_N = 0$
4. Solves for two base solutions:
   - GAMU₀: Unit freestream at α = 0°
   - GAMU₉₀: Unit freestream at α = 90°

```rust
let factorized = build_and_factorize(&geom)?;
// factorized.gamu_0: base solution for α=0°
// factorized.gamu_90: base solution for α=90°
```

### 4. Solution Combination (`solution.rs`)

Matches XFOIL's `SPECAL` and `CLCALC` subroutines:

For any angle of attack α:

```
γ_i = GAMU₀_i × cos(α) + GAMU₉₀_i × sin(α)
```

Pressure coefficient:

```
Cp = 1 - (γ / V∞)²
```

Lift coefficient (integrated pressure):

```
CL = -∮ Cp dx
```

```rust
let flow = FlowConditions::with_alpha_deg(4.0);
let solution = factorized.solve_alpha(&flow);
// solution.cl: lift coefficient
// solution.cm: moment coefficient
// solution.cp: pressure distribution
```

## Validation Against XFOIL

The implementation is validated against actual XFOIL FORTRAN output using the `Xfoil-instrumented` binary which dumps intermediate values as JSON.

### Test Results (NACA 0012, XFOIL coordinates)

When using the same coordinates as XFOIL (generated via `NACA 0012` then `SAVE`), the solver matches essentially perfectly:

| Alpha | XFOIL CL | RustFoil CL | Difference |
|-------|----------|-------------|------------|
| 0° | ~0 | 0.000 | ✓ |
| 2° | 0.2416 | 0.2416 | &lt;0.01% |
| 4° | 0.4829 | 0.4828 | &lt;0.01% |
| 6° | 0.7235 | 0.7235 | &lt;0.01% |
| 8° | 0.9634 | 0.9633 | &lt;0.01% |

**Important**: XFOIL's NACA generator adds a small trailing edge gap (~0.0025 chord) even for symmetric airfoils. Using a perfectly sharp TE (y=0 at both ends) causes numerical issues due to zero-length TE panel.

### Running Validation

To regenerate reference data from XFOIL:

```bash
cd Xfoil-instrumented/bin
rm -f fort.77
echo "NACA 0012
OPER
ALFA 0
ALFA 4
QUIT" | ./xfoil_instrumented
```

Then run tests:

```bash
cargo test -p rustfoil-inviscid --test xfoil_comparison
```

## Usage Example

```rust
use rustfoil_inviscid::{InviscidSolver, FlowConditions};

// Create solver
let solver = InviscidSolver::new();

// Load airfoil coordinates (from TE around to TE)
let points: Vec<(f64, f64)> = load_airfoil("naca0012.dat");

// Factorize once (reusable for multiple alpha)
let factorized = solver.factorize(&points)?;

// Solve at multiple angles
for alpha_deg in [0.0, 2.0, 4.0, 6.0, 8.0] {
    let flow = FlowConditions::with_alpha_deg(alpha_deg);
    let solution = factorized.solve_alpha(&flow);
    
    println!("α={:.1}°: CL={:.4}, CM={:.4}", 
             alpha_deg, solution.cl, solution.cm);
}
```

## Known Limitations

1. **Sharp TE handling**: Airfoils with perfectly sharp trailing edges (first point == last point exactly) cause numerical issues. XFOIL's NACA generator always adds a small TE gap. Use blunt TE coordinates for best results.

2. **Inviscid only**: This module computes inviscid flow. For viscous effects, use `rustfoil-bl` for boundary layer calculations.

3. **Coordinate matching**: Using the same coordinates as XFOIL gives &lt;0.01% accuracy. Different paneling (e.g., simple cosine spacing) may show larger differences.

## XFOIL Subroutine Mapping

| XFOIL Subroutine | Rust Function | Description |
|------------------|---------------|-------------|
| `NCALC` | `AirfoilGeometry::compute_normals` | Normal vectors |
| `APCALC` | `AirfoilGeometry::compute_panel_angles` | Panel angles |
| `TECALC` | `AirfoilGeometry::compute_te_geometry` | TE geometry |
| `PSILIN` | `psilin()` | Influence coefficients |
| `GGCALC` | `build_and_factorize()` | Matrix assembly |
| `SPECAL` | `FactorizedSystem::solve_alpha()` | Solution combination |
| `CLCALC` | `FactorizedSystem::compute_forces()` | Force integration |
| `STFIND` | `find_stagnation()` | Stagnation point |
